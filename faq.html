Frequently Asked Questions

How do I get started?

First download and unpack the latest client version of PAXelerate and then start PAXelerate.exe

What is the difference between the deployment version and the software developer kit (SDK)?

The deployment version is best suited for first time users and provides a quick start. The client is designed for Windows operating systems and does not require any further installations. The SDK provides full access to the source code and requires basic knowledge of the Eclipse platform. The SDK is recommended for experienced users.

How do I get started with the software developer kit (SDK)?

Make sure that Eclipse Mars.1 - Modeling Tools and Java JDK version 8 is installed. Upgrade the EMF Client Platform to version 1.7.x. Checkout the OpenCDT core plugins PAXelerate. Finally launch paxelerate.product in the plug-in net.bhl.cdt.paxelerate.core

Is there a place to find more information?

Please refere to our documentation where you can find more information about the installation, implementation as well as trouble shooting.

Under which license is PAXelerate distributed?

PAXelerate is distributed under the Eclipse Publice License (EPL). The EPL applies a weak copy left and is therefor considered to be a business friendly license.

What are the system requirements for using PAXelerate?

The current PAXelerate release only supports the Windows plaform; however, future release are targeted to be fully compatbile with MacOS and Linux as well.

What is the best way to contact support?

If this FAQ section or the documentation could not solve your problem, please use our issue tracker or use the contact form.

How can I contribute?

Contributions to the PAXelerate project are welcome by discussion or by sending in feedback, bug reports, feature proposals, models or code. The issue tracker is intended for concrete bug reports and feature requests. For further information on deployment, usage and development, please refer to the developer guidelines.
